## Overview

A hybrid recipe recommender built on the [Food.com Recipes & Interactions dataset](https://www.kaggle.com/datasets/shuyangli94/food-com-recipes-and-user-interactions/data). The system recommends recipes based on a user's rating history and filters results by what ingredients they actually have on hand.

Two collaborative filtering models are implemented **from scratch** using NumPy and SciPy (no recommendation libraries):
- **User-Based CF** — cosine similarity on mean-centered user rating vectors
- **SVD** — truncated matrix factorization via `scipy.sparse.linalg.svds`

Both are layered with an ingredient constraint system (hard and soft filtering modes) and six dietary restriction presets. A popularity baseline is included for comparison.

## Algorithms

### User-Based Collaborative Filtering

Mean-centered cosine similarity between user rating vectors. Predicted rating for user *u* on recipe *i*:

$$\hat{r}_{ui} = \bar{r}_u + \frac{\sum_{v \in N(u)} sim(u,v) \cdot (r_{vi} - \bar{r}_v)}{\sum_{v \in N(u)} |sim(u,v)|}$$

In the notebook, similarity is computed per-user on demand from the sparse matrix to avoid materializing the full 9K × 9K similarity matrix. The Streamlit app precomputes and caches it since the user-user matrix fits in memory as float32.

### SVD Matrix Factorization

Truncated SVD via `scipy.sparse.linalg.svds` on the mean-centered sparse matrix:

$$R \approx U \Sigma V^\top$$

Predictions: $\hat{r}_u = \mathbf{u}_u \cdot \Sigma \cdot V^\top + \bar{r}(u)$

Reconstructed per-user in chunks of 500 — the full predicted matrix would be ~2.7 GB.

### Popularity Baseline

Rank recipes by number of ratings in the training set. Recommend the most-rated recipes the user hasn't already rated. Non-personalized — included to check whether personalization actually helps.

### Ingredient Constraint Layer

Applied after CF scoring:
- **Hard:** keep only recipes where all ingredients are available (0 missing)
- **Soft:** keep recipes within a `max_missing` threshold, re-ranked by `(n_missing ASC, predicted_rating DESC)`

Pantry simulation: pool all ingredients from recipes the user has rated, randomly keep 70%.

## Results

### Rating Prediction (Test Set)

| Model | RMSE | MAE |
|---|---|---|
| Global Mean Baseline | 0.5996 | 0.4240 |
| SVD (k=50) | **0.5609** | **0.3544** |
| User-Based CF (k=50) | 0.6182 | 0.3660 |

SVD beats the baseline. UB-CF's RMSE is worse than the global mean — at >99% sparsity with heavy positive skew, the neighbor average adds noise. Its MAE still beats the baseline, so it captures preference direction even when magnitude is off.

### Hyperparameter Tuning

**SVD:** RMSE is stable from k=10 to k=150 (0.5609–0.5610). The first 10–20 factors capture most usable signal.

**UB-CF:** Neighborhood size 10–200 produces minimal change (RMSE 0.6314–0.6322 on a 3K test sample). The sparsity problem dominates.

### Constraint Satisfaction

| System | Candidate Coverage | Soft CSR (≤3 missing) |
|---|---|---|
| UB-CF + Hard | 0.3% | 100% |
| UB-CF + Soft | 18.4% | 100% |
| SVD + Hard | 0.8% | 100% |
| SVD + Soft | 28.7% | 100% |

Hard mode kills 99%+ of candidates. Soft with max_missing=3 is the practical choice. Returns diminish past max_missing=3.

### Top-N Quality

SVD-based systems outperform UB-CF on Precision@K and Recall@K at every list length. The popularity baseline is competitive on Precision@K due to the dataset's popularity bias, but SVD pulls ahead on Recall@K where it surfaces less obvious items.

High-activity users benefit more from personalization than low-activity users.

## Evaluation Metrics

| Metric | What it measures |
|---|---|
| **RMSE** | Root mean squared error of predicted vs actual ratings (test set) |
| **MAE** | Mean absolute error of predicted vs actual ratings |
| **Precision@K** | Fraction of top-K recommendations the user actually rated ≥ 4 |
| **Recall@K** | Fraction of liked recipes (rated ≥ 4) that appear in the top-K |
| **CSR** | Constraint Satisfaction Rate — % of recommendations satisfying the ingredient filter |

Seven systems compared: Popularity, UB-CF, SVD, UB-CF + Hard, UB-CF + Soft, SVD + Hard, SVD + Soft.
